# HWANGTODO 요구사항 정리

## 1. 제품 핵심 방향

HWANGTODO는 일반적인 TODO 앱이 아니다.  
앱을 열고 들어가서 할 일을 작성하는 앱이 아니라, **앱을 직접 클릭하지 않고도 iOS 시스템 표면에서 1초 안에 할 일을 기록하는 초고속 TODO 캡처 앱**을 목표로 한다.

핵심 컨셉은 다음과 같다.

> “앱 안에서 TODO를 쓰는 앱”이 아니라  
> “잠금화면, 액션 버튼, Siri, 단축어, 제어센터, 위젯, 알림 같은 시스템 표면에서 빠르게 기록하는 앱”

앱 본체는 할 일을 작성하는 메인 공간이 아니라, 시스템 표면에서 들어온 할 일을 나중에 정리하고, 판단하고, 일정화하고, 루틴화하고, 메모와 연결하는 **관리·정리 HQ** 역할을 해야 한다.

---

## 2. 핵심 사용자 경험

사용자는 할 일이 떠오르는 순간 앱을 열지 않아도 기록할 수 있어야 한다.

이상적인 사용 흐름은 다음과 같다.

1. 할 일이 떠오른다.
2. 잠금화면, 액션 버튼, Siri, 단축어, 제어센터, 홈 위젯, 알림 액션 등에서 즉시 기록한다.
3. 기록된 항목은 “빠른 기록” 또는 “정리 전” 영역에 저장된다.
4. 사용자는 나중에 앱을 열어 할 일을 정리한다.
5. 필요에 따라 매트릭스, 캘린더, 루틴, 메모, 나와의 채팅, 집중 모드로 연결한다.

중요한 원칙은 다음과 같다.

- 캡처 먼저, 정리는 나중에.
- 앱 실행은 선택사항이어야 한다.
- 앱보다 시스템 표면이 더 중요한 진입점이어야 한다.
- 사용자는 기록 순간에 분류, 날짜, 중요도, 우선순위를 고민하지 않아도 된다.
- 앱은 할 일 입력 화면이 아니라 정리, 검토, 계획, 실행을 위한 공간이어야 한다.

## 3. 주요 레퍼런스

### 3.1 아이젠하워 매트릭스: 투두 앱

이 앱은 제품의 중심 레퍼런스가 아니라, **정리 방식과 보조 기능의 레퍼런스**다.

참고할 요소는 다음과 같다.

- 4분면 기반 우선순위 정리
- 급함 × 중요함 기준의 판단 구조
- 매트릭스 기반 홈 위젯
- 잠금화면 위젯
- 라이브 매트릭스 / Live Activity 형태의 집중 표시
- 할 일 체크, 제거, 다음 할 일 표시
- Apple Calendar 연동
- 간단하고 직관적인 계획 구조

단, HWANGTODO의 중심은 매트릭스가 아니다.  
매트릭스는 빠르게 기록된 할 일을 나중에 정리하는 **보조 레이어**다.

### 3.2 todo mate: 할 일, 루틴 앱

todo mate는 사용자가 많은 이유를 분석해 참고해야 한다.

적용할 장점은 다음과 같다.

- 빠르고 단순한 할 일 작성
- 루틴 관리
- 오늘 해야 할 일의 명확한 시각화
- 캘린더 기반 일정 확인
- 완료감과 진행률
- 매일 쓰고 싶게 만드는 가벼운 동기부여
- 복잡하지 않은 친근한 생산성 UX
- 일기·기록처럼 느껴지는 사용성

HWANGTODO에는 todo mate의 장점을 다음 방식으로 적용한다.

- 오늘 남은 할 일 개수
- 오늘 완료율
- 루틴 완료 상태
- 캘린더에 연결된 할 일
- 간단한 일일 요약
- 완료했을 때의 만족감
- 과하지 않은 기록감

## 4. 정보 구조

권장 탭 구조는 다음과 같다.

1. 기록
2. 정리
3. 일정
4. 루틴
5. 설정

기존 용어 중 “받은함”, “보관함”은 TODO 앱 맥락에서 어색하다.  
메일 앱 같은 인상을 주기 때문에 수정이 필요하다.

권장 용어는 다음과 같다.

| 기존 용어 | 권장 용어 |
|---|---|
| 받은함 | 빠른 기록 / 기록 / 정리 전 |
| 보관함 | 완료한 일 / 지난 기록 |
| 매트릭스 | 정리 |
| 캘린더 | 일정 |
| Inbox | 빠른 기록 |
| Archive | 완료한 일 / 지난 기록 |

화면 제목은 “받은함”이 아니라 **“빠른 기록”** 또는 **“기록”**이 더 적합하다.

상단 세그먼트는 다음처럼 변경하는 것이 좋다.

- 받은함 / 보관함  
→ 정리 전 / 완료

## 5. 빠른 기록 화면 요구사항

빠른 기록 화면은 앱의 첫 화면이자 핵심 화면이다.

화면은 다음 메시지를 즉시 전달해야 한다.

> “앱을 열지 않고 남긴 할 일이 여기에 모입니다.”

필수 구성 요소는 다음과 같다.

- 큰 제목: 빠른 기록
- 부제: 앱을 열지 않고 남긴 할 일이 모이는 곳
- 빠른 입력창: 지금 떠오른 일 빠르게 남기기
- 오늘 요약
  - 오늘 할 일
  - 완료한 일
  - 정리 전 항목
  - 루틴 항목
- 출처 배지
  - Siri
  - 잠금화면
  - 액션 버튼
  - 단축어
  - 제어센터
  - 홈 위젯
  - 알림
  - 채팅
  - 앱

빠른 기록 화면은 일반적인 TODO 리스트처럼 보이면 안 된다.  
“앱 밖에서 급하게 남긴 항목이 모이는 공간”처럼 느껴져야 한다.

## 6. 시스템 표면 중심 기능

HWANGTODO의 핵심은 iOS 시스템 표면에서 할 일을 빠르게 기록하는 것이다.

필수 시스템 표면은 다음과 같다.

### 6.1 잠금화면

요구사항:

- 잠금화면 위젯 제공
- 오늘 할 일 개수 표시
- 긴급한 할 일 개수 표시
- 다음 할 일 표시
- Live Activity와 연계
- 탭하면 빠른 기록 또는 해당 할 일로 이동

주의사항:

- iOS는 잠금화면 위젯 안에서 직접 텍스트 입력을 허용하지 않는다.
- 따라서 잠금화면은 직접 입력보다는 확인, 진입, Live Activity 중심으로 설계해야 한다.
- 실제 빠른 입력은 Siri, 단축어, 액션 버튼, 알림 액션과 연결해야 한다.

권장 안내 문구:

> 잠금화면에서는 직접 글자를 입력할 수 없어요. 대신 위젯으로 바로 열거나, Siri·단축어·액션 버튼으로 앱을 열지 않고 기록할 수 있어요.

### 6.2 액션 버튼

요구사항:

- Action Button에 연결 가능한 AppIntent 제공
- 앱을 열지 않고 기록 가능해야 함
- iOS 설정에서 사용자가 단축어를 액션 버튼에 지정할 수 있도록 안내
- 기록된 항목은 빠른 기록에 저장

### 6.3 Siri

요구사항:

- Siri로 할 일 추가 가능
- 한국어 Siri 문구 제공

예시 문구:

- “HWANGTODO에 추가”
- “할 일 추가”
- “오늘 할 일 추가”
- “빠른 기록 추가”
- “나중에 정리할 일 추가”

### 6.4 단축어

요구사항:

- Shortcuts 앱에서 실행 가능한 AppIntent 제공
- 텍스트, 날짜, 메모, 분면 선택 등을 선택적으로 받을 수 있음
- 앱을 열지 않고 기록되는 흐름 우선

### 6.5 제어센터

요구사항:

- iOS 지원 범위 내에서 제어센터 컨트롤 제공
- 전체 편집기는 불가능하므로 빠른 기록 화면 또는 AppIntent로 연결
- 설정 화면에서 iOS 제약을 정직하게 안내

### 6.6 홈 화면 위젯

요구사항:

- Small / Medium / Large 위젯 제공
- 오늘 할 일 개수
- 긴급한 할 일
- 정리 전 항목
- 2×2 매트릭스 요약
- 다음 할 일
- 캘린더 연결 항목
- 루틴 진행률
- 탭 시 관련 화면으로 딥링크

### 6.7 알림 액션

요구사항:

- 할 일 알림
- 루틴 알림
- 하루 점검 알림
- 집중 알림
- 알림 액션 제공

권장 알림 액션:

- 완료
- 나중에
- 오늘로 이동
- 집중 시작
- 열기

## 7. Live Matrix / Live Activity 요구사항

Live Matrix는 잠금화면과 Dynamic Island에서 현재 집중할 일을 보여주는 기능이다.

필수 표시 요소:

- 현재 집중할 일 제목
- 분면 또는 상태
- 경과 시간 또는 집중 시간
- 다음 액션
- 진행률, 예: 1/4
- 완료 또는 다음 할 일로 이동 가능한 흐름
- 탭 시 정확한 할 일 또는 집중 화면으로 이동

주의사항:

- Dynamic Island와 Live Activity는 전체 편집기가 아니다.
- 텍스트 입력이나 복잡한 체크리스트 편집은 제한된다.
- 가능한 범위 안에서 현재 할 일 확인, 완료, 이동, 딥링크를 제공해야 한다.

## 8. 매트릭스 / 정리 기능

매트릭스는 제품의 중심이 아니라 빠르게 기록된 할 일을 정리하는 레이어다.

필수 구성:

- 4분면 구조
- 정리 전 항목을 매트릭스로 이동
- 각 분면별 할 일 개수
- 각 분면의 대표 할 일
- 분면 이동
- 완료
- 일정 잡기
- 루틴으로 만들기
- 메모 연결
- 집중 시작

권장 4분면 명칭:

| 분면 | 권장 이름 | 액션 |
|---|---|---|
| 급하고 중요함 | 지금 할 일 | 지금 하기 |
| 중요하지만 급하지 않음 | 계획할 일 | 일정 잡기 |
| 급하지만 덜 중요함 | 맡길 일 | 맡기기 |
| 급하지도 중요하지도 않음 | 줄일 일 | 줄이기 / 제거 |

기존의 딱딱한 매트릭스 용어보다 일반 사용자가 이해하기 쉬운 표현을 우선한다.

## 9. Apple Calendar 연동

Apple Calendar 연동은 실제 생산성 기능처럼 동작해야 한다.

필수 요구사항:

- 캘린더 권한 요청
- 권한 거부 상태 처리
- 권한 재요청 안내
- 할 일을 캘린더 이벤트로 생성
- 생성된 이벤트와 할 일 연결
- 오늘 일정 표시
- 예정된 일정 표시
- 지난 일정 또는 미완료 일정 표시
- 캘린더 연결 여부 표시
- 외부에서 캘린더 이벤트가 삭제된 경우 안전하게 처리

캘린더 화면은 todo mate처럼 오늘의 계획을 한눈에 볼 수 있어야 한다.

## 10. 루틴 기능

루틴은 todo mate 레퍼런스에서 중요한 요소다.

필수 요구사항:

- 루틴 생성
- 반복 주기
- 요일 선택
- 활성 / 비활성
- 오늘 루틴 표시
- 루틴 완료
- 루틴 완료율
- 루틴을 할 일과 구분해서 표시
- 할 일을 루틴으로 변환
- 루틴에 선택적으로 매트릭스 분면 지정

예시 루틴:

- 물 마시기
- 운동하기
- 출근 전 체크
- 업무일지 작성
- 퇴근 전 정리
- 캘린더 확인
- 책 읽기

루틴은 복잡한 반복 설정 앱처럼 만들지 말고, 빠르고 단순하게 만들어야 한다.

## 11. 메모 링크 / 내부 메모

각 할 일에는 메모를 연결할 수 있어야 한다.

필수 요구사항:

- 내부 메모 작성
- 외부 메모 URL 연결
- 사용자가 Apple Notes 링크를 붙여 넣으면 열 수 있게 지원
- Apple Notes의 비공개 DB에는 접근하지 않음
- 설정 또는 안내에서 이 제한을 정직하게 설명

메모는 다음 용도에 유용해야 한다.

- 업무일지
- 회의 메모
- 아이디어
- 할 일의 배경 설명
- 참고 링크

## 12. 나와의 채팅

나와의 채팅은 챗봇이 아니라 개인 생각 정리 공간이다.

목표:

- 사용자가 자연어로 생각을 적는다.
- 앱이 문장을 할 일 후보로 나눈다.
- 정리된 할 일은 빠른 기록에 저장된다.

예시:

입력:

> 오늘 업무일지 쓰고, 캘린더 확인하고, 엄마한테 전화해야 함

출력:

- 업무일지 쓰기
- 캘린더 확인하기
- 엄마한테 전화하기

주의사항:

- 실제 AI 모델이 없다면 “AI”라고 부르면 안 된다.
- deterministic parsing이면 “추천 추출”, “할 일로 정리” 같은 표현을 사용한다.
- 나와의 채팅은 챗봇 서비스처럼 보이면 안 된다.
- 개인 메모장, 생각 정리 공간처럼 느껴져야 한다.

## 13. 설정 화면 요구사항

설정 화면은 단순 설명서가 아니라 **시스템 표면 설정 체크리스트**여야 한다.

필수 항목:

- 잠금화면 위젯
- 홈 화면 위젯
- 액션 버튼
- Siri / 단축어
- 제어센터
- 알림
- 캘린더
- Live Matrix
- 메모 연결

각 항목은 다음 정보를 가져야 한다.

- 기능명
- 짧은 설명
- 현재 상태
  - 설정 필요
  - 사용 가능
  - 확인 필요
- 액션 버튼
  - 설정 방법 보기
  - 테스트하기
  - 열기

설정 화면은 “이 기능이 있습니다”가 아니라 “지금 이 기능을 사용할 수 있는지”를 보여줘야 한다.

- 받은함
- 보관함
- Archive
- Inbox
- AI 분석
- Dashboard
- Task Manager
- System Surface

권장 표현:

- 빠른 기록
- 정리 전
- 완료한 일
- 지난 기록
- 오늘 할 일
- 지금 할 일
- 일정 잡기
- 루틴으로 만들기
- 메모 연결
- 집중 시작
- 나중에 정리
- 할 일로 정리
- 앱을 열지 않고 기록

앱의 핵심 문구:

> 앱을 열지 않고 기록하세요.  
> 떠오른 일은 빠르게 남기고, 정리는 나중에 해도 괜찮아요.

## 15. 디자인 / UIUX 요구사항

디자인은 iOS 최신 스타일, 모던, 미니멀, 생산성 앱 느낌이어야 한다.

방향:

- iOS 26 스타일에 어울리는 최신 UI
- 미니멀하고 정돈된 카드
- 과하지 않은 글래스 느낌
- 넓은 여백
- 명확한 타이포 계층
- 하단 탭바와 콘텐츠가 겹치지 않음
- 다크모드 대응
- 위젯에서도 잘리지 않는 텍스트
- 시스템 앱 같은 안정감
- 데모 앱이나 샘플 앱 느낌 제거

현재 화면에서 개선이 필요한 부분:

- “받은함”이라는 제목이 컨셉을 약하게 만듦
- 설정 화면이 설명문 리스트처럼 보임
- 시스템 표면 중심 앱이라는 느낌이 아직 약함
- todo mate처럼 매일 쓰고 싶게 만드는 완료감과 루틴 UX가 약함
- 브랜드 정체성이 약함
- 전체적으로 깔끔하지만 아직 실서비스 마감 전 MVP 느낌이 있음

## 16. 구현 상태 평가 기준

기능 상태는 다음처럼 구분해서 보고해야 한다.

- Built: 코드가 빌드됨
- Wired: 기능이 연결되어 있음
- Visually Verified: 화면에서 직접 확인함
- Simulator Verified: 시뮬레이터에서 확인함
- Real Device Verified: 실기기에서 확인함
- Not Verified: 아직 확인하지 않음
- iOS Limited: iOS 정책상 불가능하거나 제한됨

“빌드 성공”만으로 완성이라고 보고하면 안 된다.

특히 별도 검증이 필요한 항목:

- 앱 실행
- 빠른 기록 저장
- Siri 기록
- 단축어 기록
- 액션 버튼 기록
- 제어센터 기록
- 홈 위젯 추가
- 잠금화면 위젯 추가
- Live Activity 표시
- Dynamic Island 표시
- 알림 액션
- Apple Calendar 이벤트 생성
- 루틴 생성 및 완료
- 메모 링크
- 나와의 채팅 → 할 일 변환

## 17. 실기기 검증 필요 항목

다음 기능은 시뮬레이터만으로 완전 검증하기 어렵다.

- Action Button
- Siri 음성 실행
- Shortcuts 실제 실행
- Lock Screen 위젯 설정
- Live Activity 잠금화면 표시
- Dynamic Island 표시
- 알림 액션
- 제어센터 컨트롤
- 실제 Apple Calendar 권한 흐름
- 위젯 갤러리 추가 경험

따라서 최종 보고서에는 반드시 실기기 검증 여부를 분리해서 작성해야 한다.

## 18. 최종 완성 기준

HWANGTODO는 다음처럼 느껴져야 한다.

- 앱을 열지 않고 할 일을 기록할 수 있다.
- 기록한 일은 한 곳에 안전하게 모인다.
- 정리는 나중에 해도 된다.
- 매트릭스는 입력 방식이 아니라 정리 방식이다.
- 캘린더로 실행 계획을 만들 수 있다.
- 루틴으로 반복 행동을 관리할 수 있다.
- 메모와 채팅으로 생각을 정리할 수 있다.
- 위젯, 잠금화면, 액션 버튼, Siri, 단축어가 실제 메인 진입점이다.
- 앱 본체는 설정, 검토, 정리, 계획을 위한 HQ다.
- 디자인은 최신 iOS 앱처럼 모던하고 미니멀하다.
- 데모 앱, 샘플 앱, 일반 TODO 앱처럼 보이면 안 된다.

## 19. 현재까지의 판단

현재까지 구현 결과는 방향은 맞게 바뀌었지만, 최종 완성으로 보기는 어렵다.

현재 상태:

- 빠른 기록 중심 구조로 전환됨
- 한국어 UI 적용됨
- 매트릭스, 캘린더, 루틴, 설정, 위젯, Live Activity, AppIntent 구조가 있음
- 앱 실행과 기본 화면은 확인됨
- 하지만 용어, 실기기 검증, 시스템 표면 UX, 설정 체크리스트, 디자인 완성도는 추가 개선 필요

현재 판단:

> 기능 뼈대는 들어갔지만, 아직 “앱을 열지 않고 1초 만에 기록하는 실서비스급 iOS 시스템 표면 TODO 앱”으로 완성되었다고 보기는 어렵다.

다음 단계는 기능 추가보다 다음에 집중해야 한다.

1. 용어 재정리
2. 빠른 기록 중심 UX 강화
3. 설정 화면을 시스템 표면 체크리스트로 개편
4. 실기기 QA
5. 위젯 / Live Activity / Siri / Action Button 검증
6. todo mate식 루틴·오늘 요약·완료감 강화
7. iOS 최신 모던 미니멀 UI 마감
