## 적대적 코드 리뷰 결과 (2026-07-08)

5개 차원(로직/동시성/API 오용/명세 준수/데이터 안전) 파인더 + 발견당 3인
반박 검증(총 77 에이전트). **확정 18건 전부 수정 완료**, 오탐 기각 6건.
주요 수정: 위젯 완료 시 잔여 알림 정리 스윕, 채팅 재변환 중복(변환 시점
제목 기록), 레거시 임포터 all-or-nothing 재구성, write-only 캘린더 정직
처리+중복 이벤트 차단, 일정 화면 이벤트/할 일 중복 표시 제거, 캡처 출처
딥링크 전달(제어센터·위젯 배지 정직화), 집중 알림 채널 신설(§6.7),
EKEventStoreChanged 디바운스+캐시, 되돌리기/지난 기록 상태 전이 알림 훅.
수정 중 발생한 SwiftData 경량 마이그레이션 파손(신규 배열 속성)은
옵셔널 속성으로 재설계해 기존 스토어 무손실 마이그레이션을 시뮬레이터에서
확인함.

## 시뮬레이터 확인 중 기록된 특이사항

- `simctl openurl`은 iOS 26 시뮬레이터에서 " 'HWANGTODO'에서 열겠습니까?" 확인
  다이얼로그를 띄운다(시뮬레이터 QA 한정 이슈). 회피용으로 DEBUG 전용
  `-hwangtodo-route <host>` 런치 인자를 제공한다. 첫 설치 직후 동일 다이얼로그가
  1회 관찰됐고(재현 불가, 시스템 아티팩트로 판정) 실기기에서 재확인 권장.

## iOS Limited (정책상 불가 — 구현 대상 아님)

- 잠금화면 위젯 내 직접 텍스트 입력 (§6.1) — 설정 화면에 정직하게 안내함
- 앱 이름 없는 Siri 호출 문구 ("할 일 추가" 단독) (§6.3)
- 제어센터 안 전체 편집기 (§6.5) — 컨트롤은 빠른 기록 진입으로 연결
- 액션 버튼/제어센터/Siri 설정 상태의 프로그램적 감지 (§13) — "확인 필요"로 표기

## 실기기 검증 체크리스트 (spec §17)

실기기(iOS 26.x, Dynamic Island는 Pro 계열)에서 순서대로:
1. 액션 버튼에 "액션 버튼으로 기록" 단축어 또는 빠른 기록 컨트롤 지정 → 잠금 상태에서 기록
2. "시리야, HWANGTODO에 추가" → 출처 배지 Siri 확인
3. 제어센터에 빠른 기록 컨트롤 추가 → 탭 → 캡처 시트
4. 잠금화면 위젯 3종 추가 → 표시/딥링크
5. 홈 위젯 3종 추가 → 인터랙티브 완료 버튼 → 앱 포그라운드 후 반영 확인
6. 집중 시작 → 잠금화면 Live Activity + Dynamic Island → 완료/다음 버튼
7. 알림 예약 → 앱 종료(스와이프킬) 상태에서 액션 탭 → 정확한 할 일 처리 확인
8. 캘린더 권한 거부→재허용 흐름, 이벤트 외부 삭제 후 "캘린더에서 삭제됨" 태그 확인
